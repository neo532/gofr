# Adding a New Protocol Transport

Each new protocol transport follows a consistent pattern: server implementation → middleware manager → registration API → generated code → integration test.

## 1. Transport server ( `transport/<name>/` )

Create a new package `transport/<name>/` with:

- **`server.go`** — `Server` struct implementing `transport.Server` (`Start(ctx)`, `Stop(ctx)`)
  - `NewServer(...)` constructor with `ServerOption` pattern
  - `Address(addr string)` option for listen address
  - `Middleware(m ...middleware.Middleware)` option for global middleware
  - `Use(m ...middleware.Middleware)` / `UseWith(method string, m ...middleware.Middleware)`
  - `Addr() string` to expose the actual listening address (for `:0` dynamic ports)
- **`middleware.go`** — `MiddlewareManager` (same pattern as `transport/http/middleware.go`)
- **`plugin.go`** — If the transport uses a plugin model (e.g. rpcx's PreCallPlugin), create an adapter
- **`server_test.go`** — Tests for registration, middleware, UseWith, lifecycle

## 2. Registration function

Add a `RegisterServiceWith` function that registers a service with the transport:

```go
func RegisterServiceWith(s *Server, serviceName string, svr interface{}) {
    // register the service with the underlying transport
}
```

## 3. Protoc plugin template ( `cmd/protoc-gen-go-svc/` )

### 3a. Create the template file

Create `<name>Template.tpl`. This template generates a `_<name>.pb.go` file.

The template should:
- Generate a wrapper adapter if needed (e.g. rpcx requires `(ctx, *args, *reply) error` signatures)
- Generate `_register<Service><NAME>` helper functions
- Generate `Register<NAME>Server(s *<name>.Server, svrs ...interface{})` — dispatches via type switch

### 3b. Embed and register the template

In **`template.go`**:
- Add `//go:embed <name>Template.tpl`
- Add `var <name>TmplContent string`
- Add `func generate<NAME>(pkg string, services []*serviceDesc) string`

### 3c. Wire up generation

In **`main.go`**:
- Add `<Name> bool` to the `protocols` struct and to `parseProtocols()`
- Add a generation block in `generateFile()` after the existing protocol blocks:

```go
if protos.<NAME> {
    <name>Filename := file.GeneratedFilenamePrefix + "_<name>.pb.go"
    ng := gen.NewGeneratedFile(<name>Filename, file.GoImportPath)
    ng.P("// Code generated by protoc-gen-go-svc. DO NOT EDIT.")
    ng.P("// source: ", file.Desc.Path())
    ng.P()
    ng.P("package ", file.GoPackageName)
    ng.P()
    <name>Output := generate<NAME>(string(file.GoPackageName), services)
    for _, line := range splitLines(<name>Output) {
        ng.P(line)
    }
}
```

## 4. Generate the example

- Install the plugin: `go install ./cmd/protoc-gen-go-svc/`
- Add the protocol name to `--go-svc_opt=protocols=...` in the protoc command
- Regenerate: `protoc ... --go-svc_opt=module=github.com/neo532/gofr,protocols=http,grpc,rpcx,openapi,<name> ...`
- The generated `_<name>.pb.go` file should appear in the api directory

## 5. Integration test

In `example/helloworld/cmd/main_test.go`:
- Add a test function `Test<NAME>Endpoint` that starts the app and makes a request
- Verify the response matches expected output

## 6. Verify

```sh
go build ./...           # builds clean
go test ./... -count=1   # all tests pass
```
